Documentação da Rota de Registro de Usuários

Descrição da Rota

A rota de registro é responsável por criar novos usuários no sistema.

Requisitos

  • O método HTTP deve ser POST.
  • O endpoint aceita apenas requisições POST.

Parâmetros

Nome Tipo Obrigatório Descrição
nickname string Sim Nicknmae do usuário.
name string Sim Nome completo do usuário.
birthdate string (formato YYYY-MM-DD) Sim Data de nascimento do usuário.
email string Sim E-mail do usuário.
password string Sim Senha do usuário.

Validações

  • Verifica se o nicknmae já existe.
  • Verifica se o e-mail já existe.
  • Valida a idade do usuário.
  • Valida o formato do e-mail.
  • Valida a força da senha.

Processo de Registro

  1. Verifica se todos os campos obrigatórios estão presentes.
  2. Verifica se o nicknmae e o e-mail já existem no banco de dados.
  3. Aplica as validações de idade, e-mail e senha.
  4. Criptografa a senha usando bcrypt.
  5. Insera o novo usuário no banco de dados.

Resposta Esperada

HTTP/1.1 201 Created
Content-Type: application/json; charset=utf-8

{
    "message": "Usuário criado com sucesso!"
}

Tratamento de Erros

  • 400 Bad Request: Campo obrigatório ausente ou valor inválido.
  • 409 Conflict: Nickname ou e-mail já existentes.
  • 500 Internal Server Error: Problema interno do servidor.

Implementação

O código-fonte da rota usa Express.js e dependências personalizadas para validação e criptografia de senhas.
