Documentação da Rota de Autenticação

Descrição da Rota

A rota de autenticação é responsável por verificar as credenciais do usuário e gerar um token JWT.

Requisitos

  • O corpo da requisição deve ser enviado como JSON.
  • O endpoint aceita apenas requisições POST.

Parâmetros

Nome Tipo Obrigatório Descrição
email string Sim E-mail do usuário.
password string Sim Senha do usuário.

Exemplo de Requisição

POST /api/v1/auth HTTP/1.1
Host: localhost:3000
Content-Type: application/json

{
    "email": "exemplo@email.com",
    "password": "senha123"
}

Resposta Esperada

HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8

{
    "message": "autenticado com sucesso!"
}

Tratamento de Erros

  • 400 Bad Request: E-mail ou senha inválidos.
  • 500 Internal Server Error: Problema interno do servidor.

Implementação

O código-fonte da rota é implementado usando Express.js e utiliza bcrypt para criptografar senhas e JWT para gerar tokens.
